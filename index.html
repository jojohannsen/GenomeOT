Genomes Over Time

At some level, genomes are incredibly stable over time. For example, chromosome 8 has a layout that has not changed in millions of years.

Species may have different numbers and sizes of chromosomes, but within those chromosomes are very large sections with a clear relationship to each other. This visualization is specific to the underlying genome re-arrangement data, which is derived from chromosome sequences in FASTA format. The underlying algorithm gives a "nearness" distance between large segments, which determines pixel color in the visualization.

In practice, this means the underlying data has a limited number of configurations that result in a clear visualization of the genome rearrangement between two species. The number of usable configurations decreases as the distance between segments grows.

The scripts here are a way to visualize relationships between species by comparing chromosomes. The input to the script is any number of FASTA chromosome files from public sites, and a configuration file listing which chromosomes to compare. The output is a set of images for each comparison.

Underlying Algorithms and Speed

Two chromosomes can be compared in linear time by creating a suffix tree for the first chromosome (linear time construction) then scanning the second chromosome counting unlikely matches in the forward or reverse direction within the suffix tree (linear time scan, linear time lookup in suffix tree). These counts represent the likelihood of common ancestry, and whether the matching section is inverted between the two species.

This underlying genomic rearrangement data is processed into a visualization, green representing forward matches, red representing reverse complement matches.

Example Configuration files

Configurations files are provided for comparing Human, Chimp, Gorilla, Pongo Abelii, and Macaca Fascicularis, along with test scripts for using those configurations.

Result Resolution

The images below are using 100,000 base pairs per pixel, and the smaller version uses 1 million base pairs per pixel. Species A is represented along the horizontal axis, Species B is represented along the vertical axis. The processing loads Species B into a suffix tree, then scans Species A in k-mer size sections (images below used 20), and a threshold for matches. Forward matches above threshold give green pixel, reverse-complement matches above threshold give a red pixel (these show the large inversions).
